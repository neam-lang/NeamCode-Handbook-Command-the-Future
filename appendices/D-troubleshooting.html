Appendix D: Troubleshooting & FAQ #

📖 10 min read

Common Issues #

Installation Problems #

"Command not found: neamcode"

Your shell can't find the NeamCode binary. Fix:

bash
# Add to PATH
$ echo 'export PATH="$HOME/.neamcode/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc

# Or for bash:
$ echo 'export PATH="$HOME/.neamcode/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc

"Error: Node.js version X is not supported"

NeamCode requires Node.js 18.0 or later:

bash
# Check your version
$ node --version

# Update via nvm
$ nvm install 20
$ nvm use 20

# Or via brew (macOS)
$ brew upgrade node

"EACCES: permission denied"

bash
$ sudo chown -R $(whoami) ~/.neamcode

Provider Problems #

"No providers available"

You need at least one AI provider configured. Quickest fix:

bash
# Install Ollama (free, local)
$ brew install ollama    # macOS
$ ollama serve &
$ ollama pull llama3.2:3b

"Missing API key for X"

Set the environment variable:

bash
$ export ANTHROPIC_API_KEY="sk-ant-..."
$ export OPENAI_API_KEY="sk-..."
$ export GEMINI_API_KEY="AI..."

"Rate limited by provider"

  • Wait a minute and try again
  • Switch to a different provider: configure a fallback in neamcode.config.yml
  • Use Ollama for non-critical tasks to reduce API calls

"Context window exceeded"

Your conversation is too long for the provider's limit:

text
You: /compact

This summarizes older messages and frees up space.

Memory Problems (v0.3.5) #

"MEMORY.md is not loading"

Check the file exists:

bash
$ ls ~/.neamcode/memory/

Each project has a subdirectory named by its SHA-256 hash. Run /memory auto in NeamCode to see the status.

"Auto memory isn't saving anything"

Auto memory extracts knowledge at session shutdown. Make sure you're:

  1. Having conversations with useful information
  2. Exiting cleanly (Ctrl+C or Ctrl+D, not killing the process)
  3. Have memory.autoMemory.enabled: true in config (default is true)

"MEMORY.md is getting too large"

The 200-line cap handles this automatically. Overflow goes to topic files. You can also:

text
You: /memory auto clear

"Old auto.json still present"

Migrate it:

text
You: /memory auto migrate

NEAMCODE.md Problems (v0.3.5) #

"My NEAMCODE.md isn't being picked up"

Check:

  1. File is named exactly NEAMCODE.md (case-sensitive)
  2. File is in the project root or a parent directory
  3. Higher-tier files may override (see 7-tier hierarchy in Chapter 11)

"@path import isn't working"

Check:

  1. The referenced file exists at the relative path
  2. Max import depth hasn't been exceeded (default: 5)
  3. Total import size hasn't been exceeded (default: 50KB)
  4. No circular imports (A imports B imports A)

"Rules in .neamcode/rules/ aren't applying"

Check:

  1. Rule files have .md extension
  2. They're in .neamcode/rules/ directory
  3. If using path-scoped rules, the YAML frontmatter paths: array matches your files
  4. Rules without paths: apply to all files (unconditional)

Offline Mode Problems (v0.3.4) #

"Offline mode isn't switching automatically"

Check:

  1. Ollama is installed and running: ollama list
  2. At least one model is downloaded: ollama pull llama3.2:3b
  3. Offline is enabled in config: offline.enabled: true

"Quality drops significantly when offline"

This is expected — local models are smaller than cloud models. Tips:

  • Use a larger model if your machine supports it (see Chapter 14)
  • Keep simpler tasks for offline, save complex ones for when you're online
  • Try qwen2.5-coder:7b for better coding quality

Performance Tips #

  1. Use /compact proactively — Don't wait for the 85% threshold. Compact when you've finished a subtask.
  1. Start new sessions for new tasks — Don't keep one session running forever. Each new task benefits from a fresh context.
  1. Use appropriate models — Don't use GPT-4o for simple file reads. Configure routing to use cheaper models for simple tasks.
  1. Background long-running tasks — Use background_shell for builds, tests, and servers so your conversation isn't blocked.
  1. Use Ollama for exploration — When you're just browsing code or asking questions, local models work great and save money.

Frequently Asked Questions #

Q: Is my code sent to the AI? A: Only the files that NeamCode (or you) explicitly reads are sent. NeamCode doesn't upload your entire codebase. Use sandbox deny patterns to protect sensitive files like .env or ~/.ssh.

Q: Can I use NeamCode without internet? A: Yes! Install Ollama and download a model. NeamCode works fully offline. See Chapter 13.

Q: Can I use NeamCode in CI/CD pipelines? A: Yes. Use neamcode --prompt "..." --output json for headless mode. Set permissions.mode: yolo in CI environments. See Chapter 24.

Q: How much does it cost? A: It depends on your provider. Ollama is free. Cloud providers charge per token. Use /cost to track spending. See Chapter 15.

Q: Can multiple people share a NeamCode session? A: Sessions are local to each user's machine. Export with /export to share conversation logs.

Q: How do I add a new AI provider? A: Add it to the providers array in neamcode.config.yml. Use type: openai-compatible for any OpenAI-compatible endpoint.

Q: What programming languages does NeamCode support? A: All of them. NeamCode works with any language. Code search and analysis features work with anything that's in a text file.

Q: How do I update NeamCode? A: Run neamcode update or reinstall from GitHub Releases.

Q: Where does auto memory store data? A: In ~/.neamcode/memory/<project-hash>/MEMORY.md. Each project gets isolated memory based on a SHA-256 hash of its git root path.

Q: How do I stop NeamCode from remembering something? A: Say "forget that X" in the chat, or run /memory auto clear to wipe all auto memory for the project.

Q: My NEAMCODE.md isn't being picked up. Why? A: Check the 7-tier hierarchy (Chapter 11). A higher-tier file may be overriding yours. The file must be named exactly NEAMCODE.md (case-sensitive).

Q: I upgraded from v0.3.4 — what about my old auto.json? A: Legacy auto.json files are auto-migrated on first startup. Run /memory auto migrate to trigger it manually.
