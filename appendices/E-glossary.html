Appendix E: Glossary #

📖 8 min read

Key terms used throughout this book, defined in plain language.

Term Definition
Agent An independent AI instance that performs a specific task. NeamCode has four built-in agents: Explore, Plan, Review, and Test. Agents have their own context and don't interfere with your main conversation.
Agentic The ability of an AI to take actions (read files, run commands, make changes) rather than just answering questions. NeamCode is "agentic" because it does things, not just talks.
Auto Memory A v0.3.5 feature where NeamCode automatically extracts and remembers useful information from your conversations, saving it to a MEMORY.md file.
Checkpoint A snapshot of your project state taken automatically before destructive operations. Use /rewind to restore a checkpoint.
Circuit Breaker A pattern that stops trying a provider after repeated failures. Prevents wasting time and money on a broken connection.
Compaction The process of summarizing older conversation messages to free up context window space. Triggered automatically at 85% capacity or manually with /compact.
Context Window The maximum amount of text (measured in tokens) that an AI provider can process at once. Ranges from 32K (Ollama) to 1M+ (Gemini).
DAP Debug Adapter Protocol. A standard protocol for debuggers. NeamCode uses DAP to debug Neam programs with breakpoints, stepping, and variable inspection.
Delegation When NeamCode spawns a sub-agent to handle a specific subtask, keeping the main conversation clean.
Fallback Chain A list of providers tried in order. If the primary fails, the secondary is tried, then the tertiary, etc.
Flap Protection Prevents rapid switching between online and offline modes. Requires sustained success/failure before switching.
Glob A file-matching pattern using wildcards. .ts matches all TypeScript files. /.test.js matches all test files recursively.
Grep Searching inside files for text matching a pattern (regular expression). Named after the Unix grep command.
LLM Large Language Model. The AI models that power NeamCode (Claude, GPT-4, Gemini, Llama, etc.).
LSP Language Server Protocol. A standard protocol for code intelligence features like autocomplete, diagnostics, and go-to-definition. NeamCode uses LSP for Neam language support.
MCP Model Context Protocol. A standard for connecting AI tools to external services and data sources. NeamCode can connect to MCP servers and also act as one.
MEMORY.md The auto-generated file where NeamCode stores persistent knowledge about your project. Located in ~/.neamcode/memory/<project-hash>/.
Mode A specialized configuration that optimizes NeamCode for a specific task type. Six modes: Explore, Debug, Test, Analyze, Deploy, Ideate.
Neam A compiled, domain-specific programming language for building AI agent systems. NeamCode is the AI terminal client for the Neam ecosystem.
NEAMCODE.md A file you create to teach NeamCode about your project. Contains rules, preferences, and context that persist across sessions. Loaded from a 7-tier hierarchy.
Ollama An open-source tool for running AI models locally on your machine. NeamCode auto-detects and uses Ollama for offline AI.
Orchestrator The core loop inside NeamCode that receives your prompt, calls the AI, executes tools, and returns results. Limited to 25 turns per prompt.
Permission Mode Controls how much autonomy NeamCode has. Four levels: safe (read-only), standard (ask before writes), power (ask only for destructive), yolo (fully automatic).
Provider An AI service that NeamCode connects to. Can be cloud-based (Anthropic, OpenAI, Gemini) or local (Ollama).
Routing Strategy How NeamCode selects which provider to use: quality (best model), speed (fastest), cost (cheapest), balanced (mix).
Rules Engine A v0.3.5 feature. Place .md files in .neamcode/rules/ to define behavior rules that apply to specific file paths or globally.
Session A saved conversation with NeamCode. Auto-saved on exit, resumable with --continue or /resume.
Skill A built-in tool that NeamCode can use to take actions. Examples: file_read, bash, git, web_search. 25+ skills are built in.
SLM Small Language Model. A compact AI model (1-7B parameters) that runs locally on consumer hardware via Ollama.
Slash Command A command starting with / that you type directly. Unlike natural language prompts, slash commands execute specific functions (e.g., /help, /mode explore, /cost).
Smart Router The system that decides which AI provider handles each request, based on the configured routing strategy and provider health.
Token A unit of text processing. Roughly 4 characters or 0.75 words in English. Used to measure context window size and API costs.
Topic Overflow When MEMORY.md exceeds 200 lines, detailed entries are moved to separate topic files (e.g., debugging.md, architecture.md).
Turn One cycle of the orchestrator: receive prompt → call AI → execute tools → return result. Maximum 25 turns per prompt.
7-Tier Hierarchy The priority system for loading NEAMCODE.md files: managed → global → ancestor → project → subdirectory → local → config. Higher numbers override lower.
@path Import A syntax in NEAMCODE.md files that inlines content from another file: @coding-standards.md inserts the contents of that file.
