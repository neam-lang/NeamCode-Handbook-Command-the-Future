Appendix B: All Skills Reference #

📖 12 min read

A complete reference of every built-in skill (tool) available in NeamCode v0.3.5.

📝 Note — You don't call skills directly. You describe what you want in natural language, and NeamCode chooses the right skills automatically. This reference helps you understand what NeamCode can do under the hood.

File Operations #

Skill Permission What It Does When It's Used
file_read Read Reads file contents from disk When you ask about a file, or NeamCode needs to understand code
file_write Write Creates or overwrites a file When you ask NeamCode to create a new file
file_edit Write Replaces specific text in a file When you ask to change part of an existing file

file_edit Details #

The file_edit skill uses precise string matching:

  • You specify an old_string (the exact text to find)
  • You specify a new_string (what to replace it with)
  • The old_string must be unique in the file
  • Whitespace and indentation must match exactly

This is why NeamCode reads a file before editing it — it needs the exact text.

Code Search #

Skill Permission What It Does When It's Used
glob Read Finds files by name pattern "Find all TypeScript files", "Where is the config?"
grep Read Searches file contents with regex "Find all uses of getUserById", "Where is TODO?"
dir_tree Read Shows directory structure "Show me the project layout"
code_structure Read Extracts classes, functions, exports "What functions are in this file?"

glob Patterns #

Pattern Matches
*.ts All TypeScript files in current dir
*/.ts All TypeScript files recursively
src/*/.test.ts All test files under src/
*.{js,ts} JavaScript and TypeScript files
config.* Any file named config with any extension

grep Parameters #

Parameter Description Example
pattern Regex to search for `"TODO\ FIXME"`
path Directory to search "src/"
glob File pattern filter "*.ts"
-i Case insensitive true
-C Context lines 3
output_mode content, files_with_matches, count "content"

Shell Execution #

Skill Permission What It Does When It's Used
bash Execute Runs a shell command "Run the tests", "Install express"
background_shell Execute Runs a command in background "Start the dev server", long-running tasks

bash Limits #

  • Default timeout: 2 minutes (120,000 ms)
  • Maximum timeout: 10 minutes (600,000 ms)
  • Interactive commands (vim, nano, less) are not supported
  • Commands requiring sudo will prompt for your password

Git Operations #

Skill Permission What It Does When It's Used
git Execute Runs git commands (status, diff, log, add, commit, etc.) Any git operation
git_pr Execute Creates pull requests via GitHub CLI "Create a PR for this branch"

Supported Git Operations #

text
status, diff, log, add, commit, push, pull,
branch, checkout, merge, stash, rebase, tag,
remote, fetch, reset, cherry-pick

⚠️ Warning — NeamCode will ask before running destructive git commands like reset --hard, push --force, or branch -D.

Web Intelligence #

Skill Permission What It Does When It's Used
web_search Read Searches the web "Search for Express.js rate limiting best practices"
web_fetch Read Fetches a URL and converts to markdown "Read the docs at this URL"

Memory #

Skill Permission What It Does When It's Used
memory_save Write Saves an entry to memory "Remember that we use PostgreSQL"
memory_list Read Lists memory entries "What do you remember about this project?"

Task Management #

Skill Permission What It Does When It's Used
task_write Write Creates/updates a task Tracking multi-step work
task_read Read Reads task details Checking task status
task_list Read Lists all tasks Reviewing work items

Agent Delegation #

Skill Permission What It Does When It's Used
delegate_to_agent Execute Spawns a sub-agent for a task Complex research, parallel work

Available Agents #

Agent Purpose
Explore Deep codebase research
Plan Implementation planning
Review Code review
Test Test writing and execution

Image Analysis #

Skill Permission What It Does When It's Used
image_read Read Analyzes images and screenshots "What does this screenshot show?"

Supports: PNG, JPG, GIF, WebP, SVG

text
