Chapter 15: Budget & Cost Tracking #

"What gets measured gets managed. What gets budgeted doesn't bankrupt."
— Every developer who got their first surprise API bill

📖 Estimated read time: 12 minutes

What You'll Learn #

  • How to set session and monthly budgets in NeamCode
  • How to use the /cost command for real-time spending visibility
  • How warning thresholds work and why 80% is the default
  • How to see cost breakdowns per provider
  • Practical strategies to reduce your AI spend without sacrificing productivity

15.1 Setting Budgets #

NeamCode gives you three levels of budget control: per-session limits, token limits, and cost ceilings.

Session Budget #

Set a maximum spend for each NeamCode session:

yaml
# neamcode.config.yml

tokens:
  session_budget: 5.00        # max $5.00 per session
  warning_threshold: 0.8      # warn at 80% ($4.00)

When you approach the limit:

text
neamcode $ Refactor the entire authentication module

⚠ Budget Warning: You've used $4.12 of your $5.00 session budget (82%).
  Remaining: $0.88
  Consider switching to a cheaper model or local provider.

Proceeding with Anthropic (claude-sonnet)...

When you hit the limit:

text
neamcode $ Now rewrite the authorization layer too

✗ Session budget exhausted: $5.00 / $5.00
  Options:
    1. Switch to local model:  /provider ollama
    2. Increase budget:        /budget 10.00
    3. Start new session:      /reset

Token Limits #

Cap the total tokens (input + output) per session:

yaml
tokens:
  max_input: 100000           # max 100K input tokens per session
  max_output: 50000           # max 50K output tokens per session
text
neamcode $ /cost

Token Usage:
  Input:   45,230 / 100,000  (45%)
  Output:  12,890 /  50,000  (26%)

Setting Budgets Mid-Session #

You can adjust budgets on the fly without editing the config file:

text
neamcode $ /budget 10.00

Session budget updated: $10.00
  Used so far: $4.12
  Remaining:   $5.88
  Warning at:  $8.00 (80%)
💡 Tip

💡 Tip: Start with a conservative budget and increase it if needed. It's easier to bump up a budget mid-session than to un-spend money you've already used.

15.2 The /cost Command #

The /cost command is your real-time spending dashboard. Run it anytime to see exactly where your money is going.

Basic Usage #

text
neamcode $ /cost

Session Cost Report
─────────────────────────────────────────
  Total cost:         $3.47
  Session budget:     $5.00
  Remaining:          $1.53
  Budget used:        69%

  Requests:           42
  Input tokens:       38,450
  Output tokens:      11,230

  Session duration:   1h 23m
  Cost per hour:      $2.51

Detailed Breakdown #

text
neamcode $ /cost --detailed

15.3 Warning Thresholds #

NeamCode warns you before you hit your budget limit, giving you time to adjust. The default threshold is 80%.

How It Works #

text
Budget:     $5.00
Threshold:  80% (default)
Warning at: $4.00

$0 ─────────────── $4.00 ──────── $5.00
                      │               │
                      ▼               ▼
                   ⚠ Warning       ✗ Budget
                   appears          exhausted

Configuring the Threshold #

yaml
tokens:
  session_budget: 5.00
  warning_threshold: 0.8      # 80% — default, recommended

You can adjust it:

yaml
  warning_threshold: 0.5      # warn at 50% — very conservative
  warning_threshold: 0.9      # warn at 90% — more headroom
  warning_threshold: 1.0      # no warning, hard stop at limit

What the Warning Looks Like #

At 80% of budget:

text
neamcode $ Explain this database schema

⚠ Budget Warning: 80% used ($4.00 / $5.00)
  Suggestion: Switch to cost routing (/routing cost) to extend your session.

[response continues normally]

At 95% of budget:

text
neamcode $ Generate API documentation

⚠ Budget Critical: 95% used ($4.75 / $5.00)
  Remaining: $0.25 (approx. 1-2 more cloud requests)
  Suggestion: Switch to local (/provider ollama) for free responses.

[response continues normally]
💡 Tip

💡 Tip: The 80% default is well-chosen. It gives you enough runway to finish your current task and decide whether to increase the budget or switch strategies.

15.4 Per-Provider Cost Breakdown #

Understanding which providers eat your budget helps you make smarter routing decisions.

text
neamcode $ /cost --by-provider

Cost by Provider:
────────────────────────────────────────────────────────
  Provider              Requests   Cost      % of Total
────────────────────────────────────────────────────────
  Anthropic (opus)            3   $2.85       68.2%
  Anthropic (sonnet)         15   $1.05       25.1%
  Anthropic (haiku)           8   $0.02        0.5%
  OpenAI (gpt-4o-mini)        4   $0.01        0.2%
  Ollama (local)             22   $0.00        0.0%
────────────────────────────────────────────────────────
  Total                      52   $3.93      100.0%

This tells a clear story: 3 premium requests consumed 68% of the budget. The 22 local requests cost nothing.

15.5 Tips to Reduce Spend #

1. Use Cost Routing as Your Default #

yaml
routing:
  strategy: "cost"

This routes everything to the cheapest available option. Most routine tasks work fine on budget models.

2. Escalate Manually for Complex Tasks #

Instead of running everything on premium, use premium only when you need it:

text
# Default: cheap/local
neamcode $ [routine coding, questions, refactoring]

# Escalate for important tasks
neamcode $ /provider anthropic claude-opus
neamcode $ Review this security-critical authentication code
neamcode $ /routing cost
# Back to cheap

3. Use Local Models for Iteration #

When you're iterating on code — writing, testing, fixing, re-testing — switch to local:

text
neamcode $ /provider ollama

# iterate freely at $0.00 cost
neamcode $ Write the function
neamcode $ Run the tests
neamcode $ Fix the failing test
neamcode $ Run again
neamcode $ Add edge case handling
neamcode $ Run tests again

# all 6 requests: $0.00

4. Be Specific in Your Prompts #

Vague prompts generate long responses (more output tokens = more cost). Specific prompts get targeted answers:

text
# Expensive (vague → long response)
neamcode $ Tell me about error handling in Python

# Cheap (specific → short response)
neamcode $ Show me try/except syntax for FileNotFoundError in Python

5. Use the Right Model Size #

Don't use Claude Opus for "what does this error mean?" and don't use phi3 for "review my distributed system design."

text
Task Complexity    Recommended Tier     Typical Cost
──────────────────────────────────────────────────────
Syntax questions   Local (free)         $0.000
Write boilerplate  Local (free)         $0.000
Simple refactoring Local or Budget      $0.000-$0.003
Test generation    Budget cloud         $0.003-$0.010
Bug analysis       Standard cloud       $0.010-$0.050
Code review        Standard cloud       $0.020-$0.080
Architecture       Premium cloud        $0.100-$0.350
Security audit     Premium cloud        $0.150-$0.500

6. Set a Budget and Stick to It #

The discipline of a budget forces better habits:

yaml
tokens:
  session_budget: 3.00         # $3 per session
  warning_threshold: 0.8

When you hit the warning, ask yourself: "Do I really need cloud for this next question, or will local work?"

7. Review Your Spending Weekly #

text
neamcode $ /cost --detailed

# Ask yourself:
# - Which premium requests were worth it?
# - Could any of them have been handled by a cheaper model?
# - What's my local ratio? Can I push it higher?
💡 Tip

💡 Tip: A good target local ratio is 70-85%. If you're below 50%, you're likely overspending on tasks that local models handle well.

15.6 Pro Tips #

💡 Tip

💡 Tip: The /cost command is free. It doesn't send anything to the AI — it's a local calculation. Check it as often as you like.

💡 Tip

💡 Tip: Budget resets each session. When you start a new NeamCode session, the cost counter resets. If you want to track monthly spending, note your session totals or check /cost --timeline at the end of each session.

💡 Tip

💡 Tip: Token warnings and cost warnings are independent. You can hit your token limit before your cost limit or vice versa. Both act as guardrails.

💡 Tip

💡 Tip: Use /budget 0 for free-only mode. Setting the budget to zero forces NeamCode to use only free (local) providers. This is an alternative to /offline if you want to stay connected but avoid spending.

text
neamcode $ /budget 0

Budget set to $0.00
  Only free providers (Ollama) will be used.
  Cloud providers disabled for this session.
💡 Tip

💡 Tip: Cost-per-hour is your best metric. The /cost command shows cost-per-hour, which is more meaningful than cost-per-request. If you're spending $2/hour on AI assistance that makes you 2x productive, that's excellent ROI.

💡 Tip

💡 Tip: Premium models have higher variance. A simple question to Claude Opus might cost $0.05. A complex one might cost $0.50. Budget accordingly — the range is wide for premium models.

⚠️ Warning

⚠️ Warning: Token counts are estimates during streaming. The final count (shown after the response completes) is accurate, but the in-progress counter may be slightly off. Don't panic if it briefly exceeds your limit — NeamCode reconciles after each response.

⚠️ Warning

⚠️ Warning: Different providers count tokens differently. An identical prompt may use 1,000 tokens on one provider and 1,100 on another due to different tokenizers. Budget with some margin.
