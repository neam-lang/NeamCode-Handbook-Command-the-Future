8.1 Getting Help #

Every journey starts with /help. When you're lost, unsure, or just curious what's available, this is your compass.

text
neamcode> /help

  NeamCode Slash Commands
  ═══════════════════════

  Session & Context
    /sessions        List saved sessions
    /resume          Resume a previous session
    /rename          Rename the current session
    /clear           Clear the conversation
    /compact         Compress context to save tokens
    /export          Export conversation to a file
    /context         Show current context usage
    /rewind          Revert to a previous checkpoint

  Modes
    /mode            Switch operating mode
    /mode explore    Understand codebases
    /mode debug      Find and fix bugs
    /mode test       Write and run tests
    /mode analyze    Data analysis tasks
    /mode deploy     Infrastructure & deployment
    /mode ideate     Brainstorm ideas

  Memory & Rules
    /memory          Manage auto memory
    /cost            Show token/cost usage

  Code & Git
    /review          Code review current changes
    /diff            Show current git diff
    /eval            Evaluate a Neam expression
    /debug-neam      Debug a Neam program

  Tools & Providers
    /models          List or switch AI models
    /mcp             Manage MCP servers
    /neam            Run Neam compiler commands
    /pkg             Manage Neam packages
    /skill           Manage skills
    /tasks           View background tasks

  Preferences
    /theme           Change UI theme
    /copy            Copy last response
    /doctor          Diagnose environment issues
    /status          Show system status
    /online          Switch to online mode
    /offline         Switch to offline mode

  Type /help <command> for details on any command.

You can also get help on a specific command:

text
neamcode> /help rewind

  /rewind — Revert to a Previous Checkpoint
  ══════════════════════════════════════════

  Usage:
    /rewind              Show available checkpoints
    /rewind <n>          Revert to checkpoint #n
    /rewind last         Revert to the most recent checkpoint

  Checkpoints are created automatically before destructive
  operations (file edits, deletes, git commands). Use /rewind
  to undo mistakes safely.
💡 Tip

💡 Tip: You don't need to memorize every command. Type / and pause — NeamCode will show autocomplete suggestions as you type.

8.2 Session Management #

Sessions are the backbone of your workflow continuity. These commands let you organize, name, resume, and clean up your work.

/sessions — List Your Saved Sessions #

text

  Saved Sessions
  ══════════════
  #   Name                    Last Active         Messages
  1   api-refactor            2 hours ago         47
  2   fix-auth-bug            Yesterday           23
  3   unnamed-session-0412    3 days ago          12
  4   deploy-staging          1 week ago          89
  5   database-migration      2 weeks ago         34

/resume — Pick Up Where You Left Off #

text
neamcode> /resume

  Select a session to resume:
  ──────────────────────────
  1. api-refactor (2 hours ago)
  2. fix-auth-bug (Yesterday)
  3. unnamed-session-0412 (3 days ago)

You can also resume by name:

text
neamcode> /resume api-refactor

  ✓ Resumed session "api-refactor" (47 messages)

text
neamcode> /rename auth-overhaul

  ✓ Session renamed to "auth-overhaul"

/clear — Start Fresh #

text

  ⚠ This will clear the current conversation.

/compact — Compress When Things Get Long #

When your conversation grows long, /compact summarizes older messages to free up context space:

text
neamcode> /compact

  Compacting conversation...
  ──────────────────────────
  Before:  142,800 tokens (71.4%)
  After:    38,200 tokens (19.1%)

  ✓ Compressed 89 messages into a summary
  ✓ Recent 12 messages preserved in full
💡 Tip

💡 Tip: NeamCode auto-compacts at 85% context usage, but you can trigger it manually anytime you feel the conversation is getting unwieldy.

8.4 Mode Commands #

Modes change how NeamCode thinks and responds. Each mode optimizes for a different kind of task.

text
neamcode> /mode

  Current mode: Explore
  Available modes:
  ────────────────
  explore    Understand codebases — reads, explains, maps
  debug      Find and fix bugs — traces, diagnoses, patches
  test       Write and run tests — generates, executes, covers
  analyze    Data analysis — queries, transforms, visualizes
  deploy     Infrastructure — builds, ships, monitors
  ideate     Brainstorm — generates ideas, alternatives, designs

Switching is instant:

text
neamcode> /mode debug

  ✓ Switched to Debug mode
  Debug mode active — I'll focus on finding and fixing bugs.
  Paste an error message or describe the issue.
📝 Note

📝 Note: Modes are covered in full detail in Chapter 9.

Command Shortcuts #

Many commands accept arguments directly, saving you an interactive menu:

text
/resume api-refactor       # Resume by name, skip the menu
/mode debug                # Switch directly, no prompt
/models ollama/llama3.1    # Switch model in one step
/export notes.md           # Export to specific file
/rewind last               # Undo the last checkpoint

Chaining Your Workflow #

A typical debugging workflow might look like:

text
neamcode> /mode debug
neamcode> /diff
neamcode> Explain why the auth middleware is failing
neamcode> /review
neamcode> /cost

Commands You'll Use Every Day #

If you're just getting started, focus on these five:

  1. /help — When you forget anything
  2. /mode — Match your tool to your task
  3. /context — Know how much room you have
  4. /cost — Keep an eye on spending
  5. /status — Quick sanity check
⚠️ Warning

⚠️ Warning: Some commands like /clear are destructive. NeamCode creates checkpoints before these operations, but it's good practice to /export important conversations before clearing them.

Exercises #

🎯 Exercise 8.1: Open NeamCode and type /help. Read through the full list. Pick three commands you haven't used before and try each one.

🎯 Exercise 8.2: Start a conversation about any topic. After a few exchanges, run /context to see your token usage. Then run /compact and check /context again. How much did it save?

🎯 Exercise 8.3: Run /doctor and check your environment. If anything shows a warning, investigate and fix it.

🎯 Exercise 8.4: Practice the session workflow: have a conversation, /rename it to something meaningful, /export it, then /clear. Start fresh, then use /resume to get back to where you were.

🎯 Exercise 8.5: Switch between at least 3 different modes using /mode. Ask the same question in each mode (e.g., "Tell me about this project's structure") and observe how the responses differ.
